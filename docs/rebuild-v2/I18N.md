# MyClipIQ v2 — i18n Strategy

## Goals
- Support EN, PT, ES without middleware complexity
- Use next-intl with minimal configuration
- No locale-prefixed routes for MVP
- Store language preference in user profile
- Translate UI only (not database records, not AI output)

## Design Decisions

### Why next-intl?
- Standard Next.js i18n library
- Server/client component support
- Message formatting
- Type-safe translations

### Why no locale routes?
- MVP simplicity
- Avoid URL complexity
- Language stored in cookie/profile
- Switcher in UI updates preference

### Language Detection Priority
1. **Logged in**: Use `profiles.language` (en/pt/es)
2. **Not logged in**: Use browser `Accept-Language` header
3. **Fallback**: `en`
4. **User override**: Language switcher updates cookie + profile

## File Structure

```

│   ├── pt.json
│   └── es.json
├── src/
│   ├── app/

│   │   └── i18n/
│   │       ├── config.ts       # Locale config (en, pt, es)
│   │       ├── request.ts      # Locale detection (server)
│   │       └── navigation.ts   # Locale-aware navigation (future)
│   └── components/
│       └── LanguageSwitcher.tsx # Dropdown to switch language
```

## Implementation Checklist

### Foundation Setup
- [ ] Install `next-intl`
- [ ] Create `messages/en.json`, `messages/pt.json`, `messages/es.json`
- [ ] Create `src/lib/i18n/config.ts`
- [ ] Create `src/lib/i18n/request.ts`
- [ ] Update `src/app/layout.tsx` to provide locale

### User Profile Integration
- [ ] Add `language` column to `profiles` table (en/pt/es, default en)
- [ ] Update profile form with language selector
- [ ] Save language preference on profile update
- [ ] Read language from profile on login

### Language Switcher
- [ ] Create `LanguageSwitcher` component (dropdown: EN/PT/ES)
- [ ] Save selection to cookie
- [ ] Save selection to profile (if logged in)
- [ ] Reload page with new locale

### Translation Coverage
- [ ] Layout: navigation, header, sidebar
- [ ] Auth: login, signup, password reset
- [ ] Dashboard: project cards, filters, status badges
- [ ] Projects: forms, buttons, status labels
- [ ] Upload: drag-drop, progress, errors
- [ ] Analysis: results, scores, suggestions
- [ ] Review: approve/reject, notes
- [ ] Notifications: templates

## What NOT to Translate (Yet)

- Database records (customer names, project names)
- AI output (hooks, captions, hashtags) — keep in source language
- File names
- Email addresses
- Technical logs

## Future Considerations

- Locale routes (`/en/dashboard`, `/pt/dashboard`) — Phase 2
- Automatic translation of AI output — Phase 3
- Currency formatting per locale — Phase 2
- Date/time formatting per timezone — Phase 2
- RTL support — Not needed (EN/PT/ES are LTR)
